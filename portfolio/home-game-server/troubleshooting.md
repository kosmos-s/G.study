# 문제 발생 및 해결 과정

## 1. 개인 PC에서 서버를 함께 실행한 문제

### 문제
친구들이 서버를 이용하려면 개인 PC를 계속 켜 두어야 했다. 본체의 조명과 팬 소음이 발생했고, 게임과 서버가 CPU·RAM을 함께 사용해 게임 성능이 저하됐다.

### 해결
저소음 미니 PC인 SER8을 중고로 구매해 게임용 PC와 서버 운영 환경을 분리했다.

### 결과
개인 PC가 꺼져 있어도 서버를 운영할 수 있게 되었고, 개인 PC에서 게임을 플레이할 때 서버로 인한 자원 경쟁이 줄었다.

---

## 2. 방에 랜선이 한 개뿐이었던 문제

### 문제
본체 컴퓨터와 서버 컴퓨터를 모두 유선으로 연결해야 했지만 거실에서 방으로 연결된 랜선은 한 개뿐이었다.

### 해결
방에 AX3000Q 공유기를 추가하고 브리지 모드로 구성해 한 개의 인입 랜선을 여러 유선 장비가 함께 사용하도록 했다.

### 결과
본체 컴퓨터와 서버 컴퓨터를 동일한 내부 네트워크에 연결하고 내부 IP를 이용한 원격 관리가 가능해졌다.

---

## 3. Windows 10의 자원 점유 문제

### 문제
Windows 10과 백그라운드 프로그램이 RAM을 사용해 게임 서버에 할당할 수 있는 메모리가 줄어들었다.

### 해결
Ubuntu Desktop 24.04.4 LTS로 운영체제를 전환했다. Linux를 처음 사용했기 때문에 CLI 전용 Ubuntu Server 대신 GUI가 포함된 Desktop 버전을 선택했다.

### 결과
Linux 환경에서 Java, SSH, UFW, Webmin, Pterodactyl 등을 직접 설치하고 관리하는 경험을 쌓았다.

---

## 4. Ubuntu 26.04 키보드 입력 지연

### 문제
처음 설치한 Ubuntu 26.04 환경에서 키보드 입력이 늦게 반응하거나 밀리는 현상이 발생했다.

### 해결
Rufus로 Ubuntu Desktop 24.04.4 LTS 부팅 USB를 다시 제작하고 운영체제를 변경했다.

### 결과
입력 문제 없이 서버를 운영할 수 있는 환경을 확보했다.

---

## 5. 원격 데스크톱 관리의 불편

### 문제
Ubuntu 원격 데스크톱으로 전체 GUI 화면을 불러오는 방식은 단순한 서버 시작·중지나 상태 확인을 하기에는 번거로웠다.

### 해결
관리 목적에 따라 도구를 분리했다.

- SSH: 프로그램 설치, 파일 이동, 방화벽 및 로그 관리
- Webmin: Ubuntu 시스템과 자원 관리
- Pterodactyl: 게임 서버 실행, 콘솔, 파일, 자원 및 백업 관리

### 결과
전체 데스크톱에 접속하지 않아도 웹 브라우저와 SSH를 통해 필요한 관리 작업을 수행할 수 있게 됐다.

---

## 6. 기본 포트가 아닌 서버의 도메인 접속

### 문제
메인 Minecraft 서바이벌 서버가 기본 포트 `25565`가 아닌 `25566`을 사용해 사용자가 주소 뒤에 포트 번호를 입력해야 했다.

### 해결
도메인 A 레코드를 공인 IP에 연결하고 SRV 레코드를 `25566` 포트로 설정했다.

### 결과
친구들이 포트 번호를 직접 입력하지 않고 도메인 주소만으로 서버에 접속할 수 있게 됐다.

---

## 7. 공유기 성능을 원인으로 의심한 사례

### 문제
Minecraft 서버 운영 중 인터넷이 버벅이는 현상이 발생했다.

### 원인 추정
기존 공유기의 처리 성능이 부족할 가능성을 의심했다.

### 해결
거실 메인 공유기와 방의 브리지 모드 공유기를 모두 업그레이드했다.

### 결과
교체 전후의 패킷 손실, 지연시간, 대역폭을 측정하지 않았고 체감 차이도 크지 않아 개선 효과를 정량적으로 입증하지 못했다.

### 배운 점
장비를 교체하기 전에 `ping`, 패킷 손실, 대역폭, 공유기 자원 사용량 등 원인 판단에 필요한 데이터를 먼저 수집해야 한다는 점을 배웠다.
