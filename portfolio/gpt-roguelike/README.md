# GPT_Roguelike

> 자동 공격과 반복 성장을 중심으로 제작 중인 Roblox 기반 3D 생존 로그라이트 프로토타입입니다.

## 프로젝트 정보

| 항목 | 내용 |
|---|---|
| 프로젝트 형태 | 개인 프로젝트 |
| 플랫폼 | Roblox |
| 개발 도구 | Roblox Studio |
| 사용 언어 | Luau |
| 장르 | 3D 생존 액션 · 로그라이트 |

| 담당 역할 | 게임 기획, 시스템 구성, 코드 통합, 테스트, 오류 분석 |

## 프로젝트를 시작한 이유

최근 사람들이 바이브 코딩으로 웹사이트, 앱, 게임 등 다양한 결과물을 만드는 모습을 보면서 저도 직접 무언가를 만들어 보고 싶다는 생각이 들었습니다. 그중 평소 관심이 있던 게임 제작에 도전해 보기로 했습니다.

게임 엔진을 선택할 때 Unity와 같은 다른 도구도 생각할 수 있었지만, 예전에 YouTube 영상을 따라 Roblox 게임을 만들어 본 경험이 있었습니다. 완전히 새로운 개발 환경을 처음부터 배우기보다 이미 한 번 사용해 본 Roblox Studio가 바이브 코딩으로 게임을 완성해 보는 데 더 적합하다고 판단했습니다.

이에 Roblox와 AI를 활용해 게임 하나를 직접 만들어 보자는 목표로 프로젝트를 시작했고, 단순한 전투 테스트에서 로비, 생존 전투, 아이템 획득, 영구 성장으로 이어지는 반복 플레이 구조를 단계적으로 추가했습니다.

## 핵심 게임 루프

```text
로비
→ 시작 무기 선택
→ 영구 능력치 강화
→ 포탈을 통해 전투 시작
→ 자동 공격으로 몬스터 처치
→ EXP와 Gold 획득
→ 전투 결과에 따라 Soul 획득
→ 로비로 복귀
→ Soul을 사용해 영구 성장
```

플레이어가 전투에서 실패하더라도 Soul과 영구 강화 요소를 통해 다음 플레이에서 더 강해질 수 있도록 구성했습니다.

## 현재 구현 상태

### 구현 완료 또는 동작 확인

| 시스템 | 구현 내용 |
|---|---|
| 로비·전투 지역 | 준비 공간과 실제 전투 공간을 분리 |
| 포탈 상호작용 | 포탈 근처에서 `E` 키를 눌러 전투 시작 |
| 몬스터 시스템 | 시간에 따라 몬스터가 생성되고 플레이어를 추적 |
| 자동 공격 | 별도의 클릭 없이 일정 주기로 적을 공격 |
| 보상 드롭 | 몬스터 처치 시 EXP 보석과 Gold 코인 생성 |
| 아이템 흡수 | 플레이어가 가까워지면 아이템을 자동으로 획득 |
| 획득 표시 | EXP와 Gold 획득량을 화면에 표시 |
| Soul 보상 | 전투 결과에 따라 영구 성장 재화 지급 |
| 영구 강화 | 최대 체력, 공격력, 시작 Gold 강화 |
| 무기 시스템 | 무기 해금과 시작 무기 선택 기능 |

### 부분 구현 또는 추가 검증 필요

| 항목 | 현재 상태 |
|---|---|
| DataStore | 저장 구조와 연동 코드를 구현했으나 테스트 환경과 실제 배포 환경에서 추가 검증 필요 |
| UI | 기능은 표시되지만 창 겹침과 디자인 통일성 개선 필요 |
| 스킬 조작 | 일부 스킬의 이동과 입력 반응 추가 점검 필요 |
| 사운드 | 일부 에셋 타입 오류가 있어 사용할 오디오 에셋 재선정 필요 |
| 시각 효과 | 공격, 피격, 사망, 레벨업 연출이 부족한 상태 |

## 시스템 구성

프로젝트는 게임 진행, 몬스터 생성, 보상, 저장 기능을 하나의 스크립트에 작성하지 않고 역할에 따라 분리하는 방향으로 구성했습니다.

```text
Workspace
├─ 로비
├─ 전투 지역
└─ 몬스터 생성 위치

ReplicatedStorage
├─ 서버·클라이언트 통신 이벤트
├─ 무기·적·플레이어 데이터
└─ 공용 에셋

ServerScriptService
├─ 게임 진행 관리
├─ 몬스터 생성
├─ 데미지 처리
├─ 보상 처리
└─ 데이터 저장

StarterGui
├─ 메인 HUD
├─ 무기·스킬 선택 UI
└─ 결과 UI
```

서버에서는 몬스터 생성, 보상 지급, 데이터 처리를 담당하고 클라이언트에서는 입력과 UI 표시를 담당하도록 구분했습니다.

세부 Explorer 구조는 현재 프로젝트 파일을 다시 확인한 뒤 실제 구조와 일치하도록 별도 기술 문서에 기록할 예정입니다.

## 주요 문제 해결 경험

### 몬스터가 로비에 생성되는 문제

**문제**  
로비와 전투 지역이 같은 Workspace 안에 존재하면서 몬스터가 전투 맵이 아닌 로비 주변에 생성되는 현상이 발생했습니다.

**확인 과정**

- 몬스터 생성 위치가 어느 폴더를 참조하는지 확인
- 로비와 전투 지역의 SpawnPoint 구분
- 전투가 시작된 상태에서만 스폰되는지 조건 점검
- 몬스터가 추적할 대상과 이동 범위 확인

**결과**  
몬스터 생성 위치와 게임 진행 상태를 별도로 관리해야 한다는 점을 확인했고, 전투 지역 전용 생성 위치를 사용하는 방향으로 구조를 수정했습니다. 일부 상황에서 문제가 다시 발생하지 않는지는 추가 플레이 테스트가 필요합니다.

### DataStore 403 오류

**문제**

```text
StudioAccessToApisNotAllowed
Cannot write to DataStore from studio
```

**확인 과정**  
처음에는 저장 코드 문제라고 생각했지만 오류 내용을 확인한 결과, Roblox Studio에서 API 서비스 접근이 비활성화되어 발생한 문제라는 것을 알게 되었습니다.

**결과**  
코드 오류와 개발 환경 설정 오류를 분리해서 확인해야 한다는 점을 배웠습니다. Studio API 접근 설정을 점검했으며, 실제 저장 유지 여부는 배포 환경에서도 추가로 검증할 예정입니다.

### 사운드 에셋 타입 오류

**문제**

```text
Asset type does not match requested type
```

**원인**  
입력한 ID가 Sound 객체에서 사용할 수 있는 오디오 에셋이 아니었습니다.

**결과**  
Roblox의 에셋 ID는 종류에 따라 사용할 수 있는 객체가 다르다는 점을 확인했고, 사운드 추가 시 에셋 타입과 사용 권한을 먼저 점검하도록 작업 순서를 변경했습니다.

## 개발 과정에서 배운 점

### 기능은 각각 동작해도 연결 과정에서 문제가 발생한다

몬스터 처치, 아이템 생성, 경험치 지급, 결과 보상과 데이터 저장은 개별 기능으로는 단순하지만 하나의 게임 루프로 연결하면 객체 생성 순서와 서버·클라이언트 통신 문제를 함께 고려해야 했습니다.

### 서버와 클라이언트 역할을 구분해야 한다

데미지와 보상처럼 조작되면 안 되는 값은 서버에서 처리하고, 입력과 화면 표시는 클라이언트에서 담당하는 구조가 필요하다는 점을 배웠습니다.

### 동작하는 기능과 재미있는 기능은 다르다

현재 자동 공격과 성장 시스템은 동작하지만 공격 이펙트와 사운드, UI 반응이 부족해 플레이 감각이 약합니다. 기능을 계속 추가하기보다 시각적 피드백과 조작감을 먼저 개선해야 한다고 판단했습니다.

### 오류 메시지를 먼저 읽는 습관이 중요하다

DataStore와 사운드 문제를 해결하는 과정에서 코드를 무작정 수정하기보다 오류 메시지, Explorer 구조, 설정값을 먼저 확인하는 것이 더 효율적이라는 점을 경험했습니다.

## 현재 한계

- 그래픽과 UI의 완성도가 낮습니다.
- 공격과 피격의 타격감이 부족합니다.
- 외부 플레이어 대상 테스트를 진행하지 않았습니다.
- 성능과 동시 접속 환경을 측정하지 않았습니다.
- DataStore가 모든 환경에서 정상 유지되는지 충분히 검증하지 못했습니다.
- 스크린샷과 플레이 영상을 포트폴리오 문서에 연결하지 않았습니다.

4. 몬스터 사망과 아이템 흡수 연출 개선

6. UI 겹침과 입력 문제 점검
7. DataStore 저장 테스트
