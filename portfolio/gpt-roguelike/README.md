# GPT Roguelike

> Roblox에서 바이브 코딩으로 제작 중인 **3D 뱀파이어 서바이벌 스타일 로그라이트** 프로젝트입니다.  
> 반복 생존 전투와 로비 성장, 직업·무기·아이템 빌드 시스템을 결합하는 것을 목표로 합니다.

## 프로젝트를 시작한 이유

최근 사람들이 바이브 코딩으로 웹사이트, 앱, 게임 등 다양한 결과물을 만드는 모습을 보면서 저도 직접 무언가를 만들어 보고 싶다는 생각이 들었습니다. 그중 평소 관심이 있던 게임 제작에 도전해 보기로 했습니다.

게임 엔진을 선택할 때 Unity와 같은 다른 도구도 고려할 수 있었지만, 예전에 YouTube 영상을 따라 Roblox 게임을 만들어 본 경험이 있었습니다. 완전히 새로운 개발 환경을 처음부터 배우기보다 이미 한 번 사용해 본 Roblox Studio가 바이브 코딩으로 게임 하나를 완성해 보는 데 더 적합하다고 판단했습니다.

이에 Roblox와 AI를 활용해 게임을 직접 만들어 보자는 목표로 프로젝트를 시작했고, 단순한 전투 테스트에서 로비, 생존 전투, 아이템 획득, 직업과 장비, 영구 성장으로 이어지는 반복 플레이 구조를 단계적으로 확장했습니다.

## 핵심 게임 루프

```text
로비에서 직업·장비 선택
→ 생존 필드 진입
→ 자동 공격과 직업별 Q/E/R 사용
→ EXP 획득 및 1/2/3 레벨업 선택
→ 무기·패시브 조합과 진화
→ 사망 또는 승리
→ Soul·장비·기록 저장
→ 로비에서 영구 성장
```

## 구현된 주요 기능

- 자동 공격 및 무기별 공격 방식
- 전사·궁수·마법사·암살자·수호자 직업
- 직업별 Q·E·R 스킬
- EXP 보석, Gold 코인, 자동 흡수
- 숫자키 1·2·3을 사용하는 레벨업 선택
- 무기와 패시브 아이템 빌드 UI
- Soul 영구 성장과 DataStore 저장 구조
- F~S 직업 무기 가챠, 확률표, 천장·조각 구조
- 인벤토리, 장착, 판매
- 원거리 적 투사체
- 랭킹 및 플레이어 거래 프로토타입
- 사망·승리 결과 처리

> 일부 기능은 구현 후 재검증이 필요한 상태입니다. 정확한 테스트 결과는 [`docs/TESTING.md`](docs/TESTING.md)에서 관리합니다.

## 기술 구조

프로젝트는 Roblox의 클라이언트·서버 구조를 기준으로 구성했습니다.

```text
ReplicatedStorage
├─ 직업·무기·아이템·적 데이터
└─ RemoteEvent와 공용 모듈

ServerScriptService
├─ 게임 진행과 몬스터 생성
├─ 데미지·보상·투사체 처리
├─ DataStore 저장
└─ 인벤토리·가챠·랭킹·거래 관리

StarterPlayerScripts
└─ HUD, 입력, 메뉴, 전투 이펙트
```

현재 맵은 `MapBuilder`가 런타임에 생성하는 프로토타입 구조입니다. 향후에는 맵과 UI는 Roblox Studio에서 제작하고, 코드만 Rojo로 관리하는 하이브리드 구조로 전환할 계획입니다.

- 진화 조건 달성 후 진화 카드 등장
- 진화 후 공격 방식과 이펙트 변경
- 전체 회귀 테스트 중 Output 오류 없음

- 핵심 기능 중 일부가 아직 재검증 단계입니다.
- UI, 맵, 사운드와 전투 타격감의 완성도가 부족합니다.
- 외부 플레이어 대상 테스트를 아직 진행하지 않았습니다.
- 장시간 플레이와 동시 접속 성능을 정량 측정하지 않았습니다.
- 저장, 거래, 결제 관련 실패 복구와 보안 검증이 더 필요합니다.
