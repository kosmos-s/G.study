# 개발 로드맵

GPT Roguelike의 다음 개발 순서를 관리합니다.

## 상태 기준

- ✅ 완료 및 동작 확인
- 🧪 구현 후 재검증 필요
- 🚧 진행 중
- 📌 계획

## 현재 상태

- **기준 버전:** `v2.7.2 Core State Fix`
- **개발 단계:** Alpha
- **현재 우선순위:** 핵심 상태 동기화와 저장 안정화
- **다음 버전:** `v2.7.3 Foundation Stability`

## 1. v2.7.3 Foundation Stability 🚧

- 직업 변경 직후 HUD 상태 갱신
- 직업 변경 직후 무기와 자동 공격 방식 갱신
- 이동속도·사거리·진화 적용 경로 통합
- Output 경고와 초기화 순서 점검
- 기존 기능 전체 회귀 테스트

## 2. 저장 안정화 📌

- 재접속 후 직업 복원
- 장착 무기와 인벤토리 복원
- Soul과 영구 강화 데이터 검증
- 저장 실패 시 재시도와 오류 기록
- 테스트 데이터와 운영 데이터 분리

## 3. 거래 안정화 📌

- 서버 권한 검증 강화
- 거래 중 취소·이탈 처리
- 아이템 중복 지급 방지
- 저장 실패 시 거래 복구 기준 마련
- 악용 가능한 입력과 요청 제한

## 4. Soul 보상 계산 수정 📌

- 사망·승리 조건별 보상 계산 확인
- 플레이 시간과 처치 수 반영 기준 정리
- 중복 지급 방지
- 결과창 표시 값과 실제 저장 값 일치 확인

## 5. Rojo 도입 📌

- 현재 Roblox Studio 중심 구조 분석
- 로컬 Rojo 프로젝트 구성
- 서버, 클라이언트, 공용 데이터 폴더 분리
- 대형 `ClientController` 역할 분리
- 게임 소스는 비공개 저장소 또는 로컬에서 관리

## 6. MapBuilder 축소 📌

- 런타임 맵 생성 범위 축소
- 로비와 생존맵을 Roblox Studio에서 직접 제작
- 코드에는 반복 생성이 필요한 오브젝트만 유지
- Terrain, Lighting, 장식 오브젝트를 Studio 중심으로 관리

## 7. Roblox다운 통합 메뉴 개선 📌

- 하단 HUD와 로비 메뉴 디자인 통일
- 직업, 인벤토리, 가챠, 랭킹, 거래 메뉴 통합
- UI 겹침과 입력 충돌 정리
- PC와 모바일 화면 대응 검토

## 8. 직업·무기·아이템·진화 완성 📌

- 직업별 역할과 스탯 차별화
- 무기별 공격 방식과 성장 곡선 조정
- 패시브 아이템 조합 확장
- 진화 조건, 카드 등장, 진화 후 공격 갱신 완성
- 가챠와 장착 무기 상태 연결 안정화

## 9. 보스 패턴 개선 📌

- 보스별 공격 예고와 회피 구간 추가
- 원거리·근거리 패턴 조합
- 보스 체력과 페이즈 UI
- 처치 보상과 다음 전투 흐름 개선

## 10. 최적화 📌

- 장시간 생존 시 적·투사체·드롭 정리
- 데미지 텍스트와 이펙트 수명 관리
- 서버와 클라이언트 처리량 점검
- 10분 이상 플레이 성능 테스트

## 11. 사운드·BGM 📌

- 피격·사망·보상·레벨업 효과음
