# 테스트 기록

이 문서는 Roblox Studio 플레이 테스트에서 직접 확인한 결과를 기록합니다.

## 표기 기준

- `[o]` 통과 또는 동작 확인
- `[x]` 실패 또는 기대한 결과와 다름
- `[ ]` 미검증 또는 추가 테스트 필요

## 최근 회귀 테스트 체크리스트

```text
[o] 로딩
[o] 최초 직업 선택
[o] 직업 변경 비용
[x] 직업 즉시 갱신
[x] 무기 즉시 갱신
[o] 직업별 Q/E/R
[o] 생존 시작
[o] 자동 공격
[o] 사거리 표시
[o] 레벨업 1/2/3
[x] 이동속도 적용
[o] 공격속도 적용
[x] 사거리 적용
[o] 흡수범위 적용
[o] 아이템 빌드 UI
[x] 진화
[o] 인벤토리 열기
[o] 장비 장착
[o] 판매
[o] 가챠
[o] 확률표
[o] 투사체
[o] 랭킹
[o] 거래
[o] 사망/승리
[ ] 재접속 저장
[o] Output 오류 없음
```

## 결과 표

| 항목 | 결과 | 비고 |
|---|---|---|
| 로딩 | 통과 | 게임 진입 확인 |
| 최초 직업 선택 | 통과 | 최초 선택 UI와 적용 흐름 확인 |
| 직업 변경 비용 | 통과 | 비용 차감 확인 |
| 직업 즉시 갱신 | 실패 | 서버 변경 후 HUD가 즉시 바뀌지 않는 경우 발생 |
| 무기 즉시 갱신 | 실패 | 직업 변경 후 자동 공격 무기가 즉시 변경되지 않음 |
| 직업별 Q/E/R | 통과 | 직업별 입력과 스킬 실행 확인 |
| 생존 시작 | 통과 | 포탈을 통한 생존 진입 확인 |
| 자동 공격 | 통과 | 클릭 없이 공격 동작 |
| 사거리 표시 | 통과 | 표시 링 노출 확인 |
| 레벨업 1/2/3 | 통과 | 숫자키 선택 확인 |
| 이동속도 적용 | 실패 | 아이템 선택 후 실제 WalkSpeed 변화 불확실 또는 미적용 |
| 공격속도 적용 | 통과 | 공격 주기 변화 확인 |
| 사거리 적용 | 실패 | UI·링·실제 공격 거리의 동기화 실패 |
| 흡수범위 적용 | 통과 | 드롭 자동 흡수 범위 변화 확인 |
| 아이템 빌드 UI | 통과 | 현재 무기·패시브 표시 확인 |
| 진화 | 실패 | 조건 달성 후 카드 또는 공격 방식 변경 실패 |
| 인벤토리 열기 | 통과 | 메뉴 진입 확인 |
| 장비 장착 | 통과 | 장비 선택과 장착 확인 |
| 판매 | 통과 | 판매 처리 확인 |
| 가챠 | 통과 | 추첨 흐름 확인 |
| 확률표 | 통과 | 확률표 UI 확인 |
| 투사체 | 통과 | 원거리 적 투사체 이동과 피격 확인 |
| 랭킹 | 통과 | 랭킹 UI와 데이터 표시 확인 |
| 거래 | 통과 | 기본 거래 흐름 확인, 안정성 검증은 별도 필요 |
| 사망/승리 | 통과 | 결과창과 종료 흐름 확인 |
| 재접속 저장 | 미검증 | 최신 패치 적용 후 재접속 복원 테스트 필요 |
| Output 오류 없음 | 통과 | 해당 테스트 세션에서 반복 오류 미발견 |

## 저장 테스트 절차

1. Studio API Services 설정 확인
2. 새 플레이어 상태에서 직업 선택
3. 무기 장착, Soul 획득, 영구 강화 진행
4. 저장이 호출되는 시점 확인
5. 테스트 종료 후 재접속
6. 직업, 무기, 인벤토리, Soul, 영구 강화 비교
7. Output에서 DataStore 오류와 중복 저장 확인

## 성능 테스트 기준

- 10분 이상 생존
- 적과 투사체 증가 시 심각한 프레임 저하가 없는지 확인
- 적, 드롭, 투사체, 데미지 텍스트, 사거리 링이 정상 정리되는지 확인
- Output에 같은 경고가 반복해서 쌓이지 않는지 확인
- 서버와 클라이언트 메모리 사용량을 같은 조건에서 비교

현재 성능은 정량 측정하지 않았으며, 위 항목은 향후 테스트 기준입니다.
