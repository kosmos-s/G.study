# Art Direction

## 핵심 컨셉

GPT Roguelike의 아트 방향은 **다크 판타지 생존 로그라이트**입니다.

3D Vampire Survivors 스타일의 대규모 생존 전투와 Hades 스타일의 로비 성장 분위기를 결합하되, 특정 게임의 화면을 그대로 복제하지 않고 Roblox 안에서 읽기 쉬운 형태로 구성합니다.

## 로비 — 어두운 마법 성소

로비는 전투 전 준비와 영구 성장을 담당하는 안전 구역입니다.

- 어두운 마법 성소
- 중앙 포탈
- Soul 영구 강화 제단
- 직업과 무기 선택 공간
- 무기 전시대
- 가챠, 인벤토리, 랭킹, 거래 메뉴 접근 지점
- 청록색 마법광과 촛불·룬 장식

로비는 메뉴를 단순히 나열하는 공간이 아니라 플레이어가 다음 생존을 준비하는 장소처럼 보이도록 제작합니다.

## 생존맵 — 숲·폐허·묘지·안개

- 황량한 숲
- 무너진 석조 폐허
- 오래된 묘지와 비석
- 낮게 깔린 마법 안개
- 전투 동선을 방해하지 않는 환경 오브젝트
- 보스 등장과 위험 구역을 강조하는 조명 변화

맵 장식은 많아 보여도 이동과 적·투사체 식별을 방해하지 않아야 합니다.

## UI 컬러

| 요소 | 방향 |
|---|---|
| 기본 패널 | 검정·남색 계열 |
| 테두리·구분선 | 어두운 회색·은색 |
| Soul | 청록 |
| Gold | 금색 |
| EXP | 보라 |
| HP | 붉은색 |
| 선택·확정 | 밝은 청록 또는 금색 |
| 위험·실패 | 붉은색 |

UI는 기능별로 따로 만들어진 창처럼 보이지 않도록 같은 패널, 여백, 제목, 버튼 규칙을 사용합니다.

## 통합 메뉴 방향

- 하단 HUD와 로비 메뉴의 스타일 통일
- 직업, 인벤토리, 가챠, 랭킹, 거래를 하나의 Roblox다운 통합 메뉴로 구성
- 화면을 과도하게 가리는 전체 화면 UI 최소화
- 숫자키와 버튼 입력을 동시에 이해할 수 있도록 표시
- PC 우선 제작 후 모바일 입력과 화면 크기 대응 검토

## 무기별 이펙트 방향

무기는 색상뿐 아니라 실루엣과 움직임으로도 구분합니다.

| 무기 | 색상 | 형태와 움직임 |
|---|---|---|
| Sword | 파랑 | 짧고 넓은 참격 |
| Spear | 청록 | 길고 얇은 관통선 |
| Dagger | 노랑 | 빠른 플래시와 연속 타격 |
| Staff | 보라 | 마법진, 구체, 범위 폭발 |
| Axe | 주황 | 무거운 궤적과 충격파 |

## 전투 피드백

- 일반 피해와 치명타 숫자 구분
- 적 피격 시 짧은 색 변화와 밀림
- 사망 시 짧은 소멸 또는 파편 연출
- EXP·Gold 흡수 궤적
- 레벨업 순간 강조
- 보스 등장 경고와 체력바
- 필요 최소한의 화면 흔들림

## 제작 방식

현재 일부 맵과 UI는 코드로 생성하지만, 최종 방향은 Roblox Studio에서 직접 배치하고 시각적으로 조정하는 방식입니다.

- 로비, 생존맵, Terrain, Lighting: Studio 제작
- 반복 생성 오브젝트와 전투 효과: 코드 관리
- UI 템플릿: StarterGui에서 시각적으로 제작
- 시스템 코드: 향후 비공개 Rojo 프로젝트에서 관리
