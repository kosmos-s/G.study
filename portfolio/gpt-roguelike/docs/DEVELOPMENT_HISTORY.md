# 개발 과정

이 문서는 실제 개발 대화와 테스트 기록을 바탕으로 정리한 프로젝트 이력입니다. 과거 커밋을 임의로 만들어내지 않고, 구현된 마일스톤과 문제 해결 과정을 기록합니다.

## 1. 시작 계기와 초기 프로토타입

사람들이 바이브 코딩으로 웹사이트, 앱, 게임 등 여러 결과물을 만드는 모습을 보면서 직접 무언가를 만들어 보고 싶다는 생각에서 시작했습니다.

그중 게임 제작을 선택했고, Unity처럼 새로운 엔진을 처음부터 배우는 대신 예전에 YouTube 영상을 따라 사용해 본 경험이 있는 Roblox Studio를 선택했습니다. 익숙한 환경과 AI를 활용해 실제로 플레이할 수 있는 게임 하나를 만들어 보는 것이 첫 목표였습니다.

- Roblox에서 로그라이크 게임 제작 목표 설정
- 3D Vampire Survivors 오마주 방향 결정
- 기본 공격, 적 추적, 피해 처리 프로토타입 제작
- `GPT_Roguelike_v0.0.1` 최초 게시

## 배운 점

1. 기능 수보다 데이터와 상태 동기화의 안정성이 중요했습니다.
2. 대형 `ClientController`는 빠른 프로토타입에는 유리하지만 유지보수 비용이 큽니다.
3. 맵 전체 런타임 생성은 초기 테스트에 빠르지만 시각적 완성도를 높이기 어렵습니다.
4. 저장·거래·결제는 서버 검증과 실패 복구가 필수입니다.
5. 테스트 체크리스트를 사용하면 회귀 버그를 빠르게 찾을 수 있습니다.
