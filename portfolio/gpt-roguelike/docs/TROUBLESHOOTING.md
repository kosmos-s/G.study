# 문제 해결 기록

GPT Roguelike 개발 중 실제로 발생한 오류와 확인 과정을 기록합니다.

확인된 원인, 수정 내용, 재검증 상태를 구분하며 테스트가 끝나지 않은 항목은 해결 완료로 표시하지 않습니다.

## 1. ProjectileManager 누락으로 맵 생성 중단

### 문제

서버 초기화 과정에서 `ProjectileManager`를 기다리거나 불러오지 못하면서 이후 Manager 연결과 `MapBuilder` 실행 흐름까지 진행되지 않았습니다.

### 확인 과정

- `ServerScriptService`에 `ProjectileManager`가 실제로 존재하는지 확인
- 코드가 기대하는 이름과 Explorer 이름 비교
- `GameManager`의 require 및 초기화 순서 확인
- 오류 이후 `MapBuilder` 호출이 실행됐는지 로그 확인

### 대응

누락된 `ProjectileManager`를 올바른 위치에 추가하고, `GameManager`가 참조하는 이름과 초기화 순서를 맞췄습니다.

### 상태

기본 초기화와 원거리 투사체 테스트는 통과했습니다. 이후 Manager 추가 시 같은 문제가 재발하지 않도록 의존성 확인이 필요합니다.

## 2. RemoteEvents 대기 오류

### 문제

`WaitForChild("RemoteEvents")` 또는 개별 RemoteEvent 참조에서 Infinite Yield 경고가 발생했습니다.

### 원인

- `RemoteEvents` 폴더 또는 이벤트가 누락됨
- 실제 Explorer 이름과 코드의 참조 이름이 다름
- 이벤트를 생성하기 전에 다른 스크립트가 먼저 기다림

### 대응

- 필요한 이벤트 목록과 Explorer 구조 비교
- 대소문자를 포함한 이름 통일
- 서버 진입점에서 이벤트 생성 후 Manager가 참조하도록 순서 정리

### 배운 점

`WaitForChild()`는 누락된 객체를 만들어 주지 않으므로, 경고가 발생하면 먼저 객체 존재 여부와 생성 순서를 확인해야 합니다.

## 3. QuestEvent OnClientEvent 누락

### 문제

서버가 `QuestEvent`를 전송했지만 클라이언트 수신 연결이 없어 RemoteEvent 큐 관련 경고가 발생했습니다.

### 원인

이벤트 생성과 서버 송신은 있었지만 `ClientController`에 대응하는 `OnClientEvent`가 없었습니다.

### 해결

클라이언트 수신 처리를 추가하고, 아직 사용하지 않는 이벤트는 불필요하게 전송하지 않도록 점검했습니다.

```text
이벤트 생성
→ 서버 송신
→ 클라이언트 OnClientEvent 수신
→ 퀘스트 상태 또는 UI 반영
```

### 상태

수신 누락 원인은 확인하고 수정했습니다.

---

## 4. ClientController Malformed string

### 문제

`ClientController` 수정 후 다음 Lua 구문 오류가 발생했습니다.

```text
Malformed string
```

### 원인

UI 문자열에 줄바꿈을 추가하는 과정에서 따옴표가 닫히지 않거나 문자열 경계가 깨졌습니다.

### 해결

오류 줄 주변의 문자열 시작·종료 문자를 다시 확인하고, 줄바꿈을 명시적인 문자열 형식으로 수정했습니다.

### 배운 점

UI, 입력, 이펙트가 한 파일에 집중된 대형 스크립트는 작은 문자열 수정도 전체 클라이언트 실행을 막을 수 있습니다. Rojo 전환 후 역할 분리가 필요합니다.

## 5. 사거리 표시가 땅 아래에 보이던 문제

### 문제

공격 사거리 링이 캐릭터 발밑 지면보다 아래에 배치되어 일부 또는 전체가 보이지 않았습니다.

### 원인

링의 Y 위치를 캐릭터 중심이나 고정 좌표 기준으로 계산하면서 실제 바닥 높이와 맞지 않았습니다.

### 해결

캐릭터의 루트 위치와 링 두께를 기준으로 Y 오프셋을 조정해 지면 위에 표시되도록 수정했습니다.

### 상태

사거리 링 표시 자체는 최근 테스트에서 통과했습니다. 실제 공격 거리와 링 크기의 동기화는 별도 재검증이 필요합니다.

---

## 6. 직업과 무기가 즉시 갱신되지 않는 문제

### 문제

직업 변경 비용은 정상 처리됐지만 HUD의 직업 정보와 자동 공격 무기가 즉시 변경되지 않았습니다.

### 원인 추정

서버 데이터는 변경됐지만 클라이언트가 최신 전체 상태를 다시 받지 못했거나, 기존 자동 공격 상태가 남아 있었을 가능성이 있습니다.

### 수정 방향

```text
클라이언트 직업 변경 요청
→ 서버 비용·조건 검증
→ 서버 직업과 장착 무기 변경
→ 변경된 전체 상태 전달
→ HUD와 자동 공격 재구성
```

### 상태

`v2.7.2 Core State Fix`에서 수정했으나 최근 체크리스트에서는 직업·무기 즉시 갱신이 실패했습니다. `v2.7.3`에서 재검증합니다.

## 7. 이동속도·사거리·진화 적용 문제

### 문제

아이템 선택과 UI 표시는 동작하지만 실제 게임 계산에 적용되지 않는 항목이 있었습니다.

- 이동속도 선택 후 `Humanoid.WalkSpeed`
- 사거리 선택 후 실제 공격 거리
- 사거리 링 크기와 실제 공격 판정
- 조건 달성 후 진화 카드 등장
- 진화 후 무기와 공격 방식 교체

### 원인 추정

아이템 데이터, 서버 저장 상태, 클라이언트 표시, 실제 전투 계산이 서로 다른 값을 사용했을 가능성이 있습니다. 진화는 데이터 ID, 요구 패시브, 무기 레벨, 후보 생성, 무기 교체 흐름 중 하나 이상이 연결되지 않았을 수 있습니다.

### 수정 방향

- 하나의 최종 스탯 계산 결과를 이동·공격·HUD·링에 공통 사용
- 진화 조건을 서버에서 검증
- 진화 후보 생성 후 기존 무기를 명확히 교체
- 무기 교체 결과를 클라이언트에 다시 전달

### 상태

`v2.7.2`에서 수정 시도했으나 최근 테스트에서 이동속도, 사거리, 진화가 실패했습니다. 해결 완료가 아닌 재검증 상태입니다.

## 8. DataStore Studio API 설정 문제

### 문제

Studio 테스트에서 다음 오류가 발생하며 데이터를 저장할 수 없었습니다.

```text
DataStoreService: StudioAccessToApisNotAllowed
Cannot write to DataStore from studio
```

### 원인

Roblox Studio의 API 서비스 접근 설정이 비활성화된 상태였습니다.

### 해결

```text
Game Settings
→ Security
→ Enable Studio Access to API Services
```

설정을 활성화한 뒤 Studio에서 DataStore 접근을 허용했습니다.

### 상태

환경 설정 문제는 해결했습니다. 재접속 후 직업, 무기, 인벤토리, Soul, 영구 강화가 정확히 복원되는지는 별도 테스트가 필요합니다.

## 9. 몬스터가 로비에 생성되는 문제

### 문제

생존 전투가 시작되기 전이거나 잘못된 SpawnPoint를 참조할 때 몬스터가 로비 주변에 생성됐습니다.

### 대응

로비와 생존맵의 생성 위치를 분리하고, 생존 세션이 시작된 상태에서만 전투 지역 SpawnPoint를 사용하도록 수정했습니다.

### 상태

기본 테스트에서는 분리됐으며 전체 회귀 테스트에서 재발 여부를 확인합니다.

---

## 10. 사운드 에셋 타입 불일치

### 문제

```text
Asset type does not match requested type
```

Sound 객체에 지정한 ID가 오디오 에셋이 아니거나 사용 권한이 맞지 않았습니다.

### 대응

에셋 타입, 공개 범위, 사용 권한, `SoundId` 형식을 확인하고 문제가 있는 임시 사운드를 제거했습니다. 최종 사운드와 BGM은 향후 작업으로 남아 있습니다.

## 문제 해결 원칙

1. 오류 메시지와 발생 줄을 먼저 확인합니다.
2. 코드 수정 전에 Explorer 구조와 객체 이름을 비교합니다.
3. 초기화 순서와 이벤트 송신·수신을 한 흐름으로 확인합니다.
4. 서버 상태 변경과 클라이언트 UI 갱신을 별도로 확인합니다.
5. 구현 성공과 플레이 테스트 성공을 구분합니다.
6. 수정한 기능과 기존 기능을 함께 회귀 테스트합니다.
7. 확인되지 않은 원인은 추정으로 기록하고 테스트 후 확정합니다.
